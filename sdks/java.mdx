The `io.github.longbridge:openapi-sdk` Maven artifact provides a Java interface to Longbridge OpenAPI. It requires Java 11 or later. All I/O methods return `CompletableFuture` — call `.get()` to block until the result is ready, or compose with `.thenAccept` and similar methods.

<Info>
  Full API reference is available at [longbridge.github.io/openapi/java](https://longbridge.github.io/openapi/java/index.html).
</Info>

## Setup

<Steps>
  <Step title="Add the dependency">
    Add the artifact to your `pom.xml`:

    ```xml pom.xml
    <dependencies>
        <dependency>
            <groupId>io.github.longbridge</groupId>
            <artifactId>openapi-sdk</artifactId>
            <version>LATEST</version>
        </dependency>
    </dependencies>
    ```
  </Step>

  <Step title="Authenticate">
    The SDK supports two methods. OAuth 2.0 is recommended — no HMAC signatures required and tokens refresh automatically.

    **OAuth 2.0**

    Register an OAuth client to obtain a `client_id`:

    <CodeGroup>

    ```bash macOS / Linux
    curl -X POST https://openapi.longbridge.com/oauth2/register \
      -H "Content-Type: application/json" \
      -d '{
        "client_name": "My Application",
        "redirect_uris": ["http://localhost:60355/callback"],
        "grant_types": ["authorization_code", "refresh_token"]
      }'
    ```

    ```powershell Windows
    Invoke-RestMethod -Method Post -Uri https://openapi.longbridge.com/oauth2/register `
      -ContentType "application/json" `
      -Body '{
        "client_name": "My Application",
        "redirect_uris": ["http://localhost:60355/callback"],
        "grant_types": ["authorization_code", "refresh_token"]
      }'
    ```

    </CodeGroup>

    The response contains your `client_id`. Pass it to `OAuthBuilder`. On first run the SDK opens a browser authorization page; on subsequent runs it loads and refreshes the cached token from `~/.longbridge/openapi/tokens/<client_id>` automatically.

    ```java
    import com.longbridge.*;

    class Main {
        public static void main(String[] args) throws Exception {
            try (OAuth oauth = new OAuthBuilder("your-client-id")
                    .build(url -> System.out.println("Open this URL to authorize: " + url))
                    .get();
                 Config config = Config.fromOAuth(oauth)) {
                // Use config to create contexts...
            }
        }
    }
    ```

    **Legacy API key**

    Set three environment variables from your Longbridge user center:

    <CodeGroup>

    ```bash macOS / Linux
    export LONGBRIDGE_APP_KEY="App Key get from user center"
    export LONGBRIDGE_APP_SECRET="App Secret get from user center"
    export LONGBRIDGE_ACCESS_TOKEN="Access Token get from user center"
    ```

    ```powershell Windows
    setx LONGBRIDGE_APP_KEY "App Key get from user center"
    setx LONGBRIDGE_APP_SECRET "App Secret get from user center"
    setx LONGBRIDGE_ACCESS_TOKEN "Access Token get from user center"
    ```

    </CodeGroup>
  </Step>
</Steps>

## Quote API — fetch quotes

`QuoteContext.getQuote` returns a `CompletableFuture<SecurityQuote[]>`. Call `.get()` to block for the result.

```java
import com.longbridge.*;
import com.longbridge.quote.*;

class Main {
    public static void main(String[] args) throws Exception {
        try (OAuth oauth = new OAuthBuilder("your-client-id")
                .build(url -> System.out.println("Open this URL to authorize: " + url))
                .get();
             Config config = Config.fromOAuth(oauth);
             QuoteContext ctx = QuoteContext.create(config)) {
            SecurityQuote[] resp = ctx.getQuote(
                    new String[] { "700.HK", "AAPL.US", "TSLA.US", "NFLX.US" }).get();
            for (SecurityQuote obj : resp) {
                System.out.println(obj);
            }
        }
    }
}
```

## Quote API — subscribe to push events

Register a callback with `setOnQuote`, call `subscribe`, then block the thread to keep the process alive and receive events.

```java
import com.longbridge.*;
import com.longbridge.quote.*;

class Main {
    public static void main(String[] args) throws Exception {
        try (OAuth oauth = new OAuthBuilder("your-client-id")
                .build(url -> System.out.println("Open this URL to authorize: " + url))
                .get();
             Config config = Config.fromOAuth(oauth);
             QuoteContext ctx = QuoteContext.create(config)) {
            ctx.setOnQuote((symbol, quote) -> System.out.printf("%s\t%s\n", symbol, quote));
            ctx.subscribe(
                    new String[] { "700.HK", "AAPL.US", "TSLA.US", "NFLX.US" },
                    SubFlags.Quote).get();
            Thread.sleep(30000);
        }
    }
}
```

<Note>
  `Thread.sleep` keeps the main thread alive so the callback thread can deliver push events. Without it the `try-with-resources` block closes the context before any events arrive.
</Note>

## Trade API — submit an order

```java
import com.longbridge.*;
import com.longbridge.trade.*;
import java.math.BigDecimal;

class Main {
    public static void main(String[] args) throws Exception {
        try (OAuth oauth = new OAuthBuilder("your-client-id")
                .build(url -> System.out.println("Open this URL to authorize: " + url))
                .get();
             Config config = Config.fromOAuth(oauth);
             TradeContext ctx = TradeContext.create(config)) {
            SubmitOrderOptions opts = new SubmitOrderOptions("700.HK",
                    OrderType.LO,
                    OrderSide.Buy,
                    200,
                    TimeInForceType.Day).setSubmittedPrice(new BigDecimal(50));
            SubmitOrderResponse resp = ctx.submitOrder(opts).get();
            System.out.println(resp);
        }
    }
}
```

<Note>
  All context methods return `CompletableFuture`. Call `.get()` to block for the result, or use `.thenAccept`, `.thenApply`, and similar methods for non-blocking composition.
</Note>

## Environment variable reference

| Variable | Description |
|----------|-------------|
| `LONGBRIDGE_APP_KEY` | App Key from user center |
| `LONGBRIDGE_APP_SECRET` | App Secret from user center |
| `LONGBRIDGE_ACCESS_TOKEN` | Access Token from user center |
| `LONGBRIDGE_LANGUAGE` | Response language: `zh-CN`, `zh-HK`, or `en` (default: `en`) |
| `LONGBRIDGE_HTTP_URL` | HTTP endpoint (default: `https://openapi.longbridge.com`) |
| `LONGBRIDGE_QUOTE_WS_URL` | Quote WebSocket endpoint (default: `wss://openapi-quote.longbridge.com/v2`) |
| `LONGBRIDGE_TRADE_WS_URL` | Trade WebSocket endpoint (default: `wss://openapi-trade.longbridge.com/v2`) |
| `LONGBRIDGE_ENABLE_OVERNIGHT` | Enable overnight quotes: `true` or `false` (default: `false`) |
| `LONGBRIDGE_PUSH_CANDLESTICK_MODE` | Candlestick push mode: `realtime` or `confirmed` (default: `realtime`) |
| `LONGBRIDGE_LOG_PATH` | Path for SDK log files (default: no logs) |

## Troubleshooting

- On Windows, `setx` requires opening a new terminal. Use `set` to apply variables in the current `cmd.exe` session.
- If you do not see push events, ensure the main thread stays alive with `Thread.sleep` or similar.
- Set `LONGBRIDGE_LOG_PATH` to a writable directory to enable SDK debug logs.
